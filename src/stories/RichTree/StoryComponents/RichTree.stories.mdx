# RichTree

`RichTree` is an advanced, highly extensible tree view component built on **MUI X RichTreeViewPro**.

It is designed for **large interactive hierarchies** with features like:

- Drag-and-drop aware UI layer (implementation-agnostic)
- Node customization with icons, buttons, and styles
- Separation of domain data from presentation logic
- Scalable performance for deeply nested trees

---

## Core Architecture

### Resolver-Based Design

At its core, `RichTree` uses a **Prop Resolver system**:

- UI properties are computed **at render time** instead of being stored on data nodes
- Domain data (`items`) remain **UI-agnostic**
- Resolvers are **stable and memoized**, ensuring minimal re-renders
- Each node can be customized independently (icons, buttons, styles, dropzones)

This pattern ensures **referential stability**, centralized UI logic, and **O(1) lookups** for node depth.

---

### Node Highlights & Search Support

The `highlightedLabelSegment` prop provides **search-aware highlighting** within node labels. This is purely visual and does **not modify the underlying data**.

---

### Drag-and-Drop Awareness

The component is **DnD-aware** but **decoupled from any DnD library**:

- Dropzones are selectively enabled per node - you control which nodes display drop targets
- Actual drag/drop logic is delegated to the host app
- Enables **flexible integration** without locking the component to a specific DnD solution

---

### Performance Optimizations

- **Depth Map Memoization:** precomputes node depth for O(1) lookups
- **Stable Resolver References:** avoids unnecessary re-renders
- **Flat Metadata Lookup:** enables large, nested trees to remain responsive
- **Pure Hooks:** minimal side effects, consistent UI behavior

### Usage Highlights

- **End icon buttons** appear on applicable nodes only
- **Start icons** can be customized per node
- **Component never mutates your original data**
- Each node can have independent icons, buttons, styles, or dropzone behavior

---

### Technical Highlights

- **Data-Pure Architecture:** keeps domain model and UI logic separate
- **Highly Extensible:** slot overrides + resolver pattern
- **DnD-Aware:** UI responds to drag/drop state without binding to any library
- **Performance-Optimized:** depth map, stable references, memoized resolvers
- **React-Friendly:** minimal re-renders, pure hooks, predictable behavior
